# V0.2 Development Plan: Desktop Enhancements + DeepSeek LLM

## Summary

V0.2 continues the local desktop workflow direction. It focuses on making the current desktop app reliable for IELTS speaking review: local media import, playback, timestamped transcript interaction, editable transcript segments, structured IELTS scorecards, local text-only review history, editable feedback, and copy output.

This version does not implement Chrome extension workflows, browser media sniffing, webpage autofill, RAG, cloud sync, signing, notarization, auto-update, or multi-user accounts.

DeepSeek is added as an LLM feedback provider only. ASR remains supported through Mock, OpenAI, Groq, and NVIDIA. DeepSeek's official API is OpenAI-compatible Chat Completions at `https://api.deepseek.com/chat/completions`; the available V0.2 models are `deepseek-v4-flash` and `deepseek-v4-pro`.

Official references:

- https://api-docs.deepseek.com/
- https://api-docs.deepseek.com/api/create-chat-completion
- https://api-docs.deepseek.com/quick_start/pricing/

## Implementation Scope

- Add `deepseek` to the LLM provider type, settings model, model dropdown, key readiness checks, secure key storage, and provider facade.
- Keep DeepSeek out of the ASR provider dropdown.
- Store DeepSeek API keys through the existing Tauri keyring command path; do not write API keys to browser localStorage.
- Permit desktop HTTP requests to `https://api.deepseek.com/*` in the Tauri capability file.
- Use `deepseek-v4-flash` as the default DeepSeek model and `deepseek-v4-pro` as the optional higher-capability model.
- Use the existing IELTS JSON feedback prompt, streaming SSE parser, scorecard parser, and editable final feedback workflow.
- For DeepSeek feedback requests, include `response_format: { "type": "json_object" }`, `stream: true`, `temperature: 0.3`, and `max_tokens: 384000`, matching the documented maximum output for DeepSeek V4.
- For NVIDIA LLM feedback requests, use `max_tokens: 16384`, matching the documented `deepseek-ai/deepseek-v4-flash` endpoint range/default.

## Validation Plan

- Automated checks: `npm.cmd run test`, `npm.cmd run typecheck`, `npm.cmd run build`, and `cargo check`.
- Unit coverage: DeepSeek model defaults, LLM readiness/key checks, connection-test endpoint, streaming feedback request body, and OpenAI-compatible SSE parsing.
- Live smoke: read `D:\CodexProject\testResource\deepseekApiKey.txt` into memory, call DeepSeek with `deepseek-v4-flash`, and verify a minimal response without printing or storing the key.
- App-path smoke: use a short transcript through the provider adapter and verify the generated payload parses into a scorecard and feedback markdown.
- Desktop manual checks: select DeepSeek in Settings, save the key, test connection, generate feedback from a transcript, edit/copy feedback, and verify history save/reopen/search/delete.

## Test Resources

The local test resource directory is `D:\CodexProject\testResource`.

- `deepseekApiKey.txt`: local DeepSeek API key for smoke testing. Never print, commit, or copy this value into logs or docs.
- `nvApiKey.txt`: local NVIDIA API key for existing provider smoke tests. Never print, commit, or copy this value into logs or docs.
- `speakTest-nvidia-asr.wav`: short audio sample suitable for NVIDIA ASR fallback validation.
- `speakTest.mp4`: oversized media sample suitable for limit-state checks.
- Existing `*.local.txt` and `*.local.md` files are saved non-secret provider outputs for comparison.

## Assumptions

- V0.2 is a runnable development build target, not a signed distributable release.
- Real provider verification is performed when local API keys are available; missing keys are recorded as skipped rather than blocking local development.
- API keys are treated as secrets even when stored in local test resources.
- Deleting review history remains one explicit review file at a time; no bulk file or directory deletion is part of V0.2.
